# 요구사항: 폴짝 MVP

## 개요

AI가 디자이너·개발자의 PDF 포트폴리오를 분석해 개선점을 제안하고,
사용자 간 피드백·프로젝트·스터디 모집을 연결하는 커뮤니티를 제공하는 웹 서비스.
팀원 시연을 목표로 하는 MVP이며, AI 개선과 커뮤니티 두 도메인으로 구성된다.

---

## 도메인 1: AI 포트폴리오 개선

### 사용자 스토리

- As a **취업 준비 중인 개발자/디자이너**, I want to **PDF 포트폴리오를 업로드하고 AI 분석을 받고 싶다**, So that **내 포트폴리오의 약점을 파악하고 개선할 수 있다**.
- As a **사용자**, I want to **"이런 방향으로 개선하고 싶다"는 추가 텍스트를 입력할 수 있다**, So that **AI가 내 의도에 맞는 맞춤 피드백을 줄 수 있다**.
- As a **사용자**, I want to **분석 결과를 저장하고 수정 후 재업로드해 재분석받을 수 있다**, So that **개선 과정을 반복적으로 검증할 수 있다**.

### 기능 요구사항

- FR-1: 사용자는 PDF 파일만 업로드할 수 있다. 다른 형식은 업로드 전 차단하고 안내 메시지를 표시한다.
- FR-2: 파일 제한 조건 — 용량 10MB 이하, 페이지 수 최대 50페이지. 두 조건 모두 업로드 UI에 미리 표시한다.
- FR-3: 추가 텍스트 입력란(선택)을 제공한다. 미입력 시 AI가 전체적으로 분석한다.
- FR-4: AI 분석 결과는 세 가지 형식을 조합해 제공한다:
  - 항목별 점수 + 코멘트 (예: "프로젝트 설명 명확성 7/10")
  - 자유 텍스트 종합 피드백
  - 개선 문장 제안 (Before / After)
- FR-5: 분석 결과는 계정에 영구 저장되며 마이페이지에서 다시 열람할 수 있다.
- FR-6: 포트폴리오를 수정 후 재업로드해 재분석을 받을 수 있다. 이전 결과와 구분되어 저장된다.
- FR-7: 분석 결과를 PDF 다운로드 또는 공유 링크로 내보낼 수 있다.
- FR-8: 공유 링크는 비로그인 사용자도 열람 가능하다. 작성자 이름이 노출된다. 좋아요·댓글은 로그인 필요.
- FR-9: 작성자는 공유 링크를 언제든 비공개로 전환할 수 있다.
- FR-10: 분석 결과를 커뮤니티 게시글로 바로 공유할 수 있다.

### 에러 처리 요구사항

- FR-11: 이미지 스캔 PDF로 텍스트 추출이 불가능한 경우, 사용자에게 명확히 안내하고 재업로드를 유도한다.
- FR-12: 포트폴리오가 아닌 문서를 올릴 경우 차단하지 않고, 분석 결과를 그대로 노출한다 (AI가 판단한 최선의 결과).
- FR-13: 모든 에러(용량 초과, 페이지 초과, 형식 오류, 분석 실패 등)는 사용자에게 명확한 메시지로 표시한다.

## 도메인 2: 커뮤니티

### 사용자 스토리

- As a **사용자**, I want to **내 AI 분석 결과를 커뮤니티에 공유하고 사람들의 추가 피드백을 받고 싶다**, So that **AI 외에 실제 사람의 시각도 얻을 수 있다**.
- As a **사용자**, I want to **프로젝트·스터디 모집 글을 올리고 싶다**, So that **함께할 팀원이나 스터디원을 찾을 수 있다**.
- As a **사용자**, I want to **관심 있는 글의 작성자에게 1:1 DM을 보낼 수 있다**, So that **모집 글에 참여 의사를 직접 전달할 수 있다**.

### 접근 권한 정책

| 기능                                            | 비로그인       | 로그인 |
| ----------------------------------------------- | -------------- | ------ |
| 프로젝트·스터디 모집 탭 — 게시글 목록·상세 열람 | ✅             | ✅     |
| 프로젝트·스터디 모집 탭 — 댓글 읽기             | ✅             | ✅     |
| 프로젝트·스터디 모집 탭 — 좋아요 카운트 보기    | ✅             | ✅     |
| 피드백 탭 — 목록 열람                           | ✅             | ✅     |
| 피드백 탭 — 상세 열람                           | ❌ 로그인 모달 | ✅     |
| 게시글 작성                                     | ❌ 로그인 모달 | ✅     |
| 댓글 작성                                       | ❌ 로그인 모달 | ✅     |
| 좋아요 누르기                                   | ❌ 로그인 모달 | ✅     |
| 1:1 DM                                          | ❌ 로그인 모달 | ✅     |

> **UX 패턴**: 비로그인 사용자가 쓰기 액션 시도 시 페이지 이동 없이 로그인 모달을 띄운다.
> "로그인하고 좋아요 누르기" 등 CTA를 모달에 포함해 자연스럽게 유도한다.

### 기능 요구사항

- FR-14: 커뮤니티는 세 개의 카테고리 탭으로 구성된다:
  - **피드백** — AI 분석 결과 공유 및 추가 피드백 요청 (로그인 필요)
  - **프로젝트 모집** — 프로젝트 팀원 모집 글
  - **스터디 모집** — 스터디 모집 글
- FR-14-1: 세 게시판은 동일한 게시글 구조(제목 + 본문)를 공유하며, 카테고리 필드로만 구분한다. 게시판별 전용 필드는 없다.
- FR-15: 게시글 작성, 댓글, 좋아요 기능을 제공한다.
- FR-15-1: 게시글 본문은 평문 텍스트(textarea)만 지원한다. 최대 5000자, 줄바꿈 지원.
- FR-15-2: 본문에 포함된 외부 링크(http/https)는 자동으로 클릭 가능한 링크로 변환한다. `javascript:` 등 위험 프로토콜은 차단한다.
- FR-15-3: 평문 텍스트도 XSS 방지를 위해 서버·클라이언트 양쪽에서 escape 처리한다.
- FR-15-4: 게시글에 이미지를 첨부할 수 있다.
  - 형식: jpg, png, webp, gif
  - 장당 최대 5MB
  - 게시글당 최대 10장
  - 저장소: Supabase Storage
- FR-16: 피드백 탭 게시글 작성 시 AI 분석 결과를 첨부할 수 있다.
- FR-17: 사용자는 게시글 작성자에게 1:1 DM을 보낼 수 있다.
- FR-18: 1:1 채팅은 실시간으로 동작한다 (Supabase Realtime).
  - 메시지 송수신 및 실시간 표시
  - 채팅 목록 (내가 참여 중인 대화방 목록)
- FR-19: 댓글, 좋아요, DM 수신 시 웹 내 실시간 알림을 제공한다.
- FR-20: 게시글·댓글 신고 기능을 제공한다. MVP에서는 신고 내용을 DB에 저장하고, 처리는 운영자가 DB 직접 조작으로 수행한다.

### 채팅 정책 (카카오톡 스타일)

- 활성 채팅 내역은 무제한 보관한다.
- 본인 메시지 삭제 가능. 상대방에게는 "삭제된 메시지"로 표시된다.
- 대화방 나가기는 본인 화면에서만 사라지고, 상대방 화면에는 유지된다.

## 데이터 보존 정책

### 활성 사용자

- AI 분석 결과: 무제한 보관
- 채팅 내역: 무제한 보관

### 회원 탈퇴 시

- 탈퇴 후 30일 유예 기간 (계정 복구 가능)
- 30일 경과 후 개인정보 영구 삭제
- 게시글·댓글: 내용 유지, 작성자명 → "탈퇴한 사용자"로 표시
- AI 분석 결과: 영구 삭제
- 채팅 메시지: 보존, 작성자명 → "탈퇴한 사용자"로 표시

## 비기능 요구사항

- NFR-1 (보안): 모든 API 입력값은 Zod로 검증한다. 인증 필요 라우트는 미들웨어로 일괄 처리한다.
- NFR-2 (인증): 카카오·구글 OAuth를 지원한다 (이메일/비밀번호 회원가입 X).
  - OAuth 제공 이메일을 받지 못하는 경우 닉네임만으로 가입을 진행한다.
- NFR-3 (반응형): 데스크탑·모바일 양쪽에서 사용 가능한 반응형 UI를 제공한다.
- NFR-4 (성능): AI 분석은 스트리밍 응답으로 진행 상태를 사용자에게 실시간으로 보여준다.
- NFR-5 (파일): PDF 전용, 최대 10MB, 최대 50페이지. 업로드 UI에 미리 공지한다.
- NFR-6 (신고): MVP에서는 신고를 DB에 저장한다. 관리자 페이지는 MVP 이후 구현한다.
- NFR-7 (프로필): 사용자는 마이페이지에서 프로필 사진 1장을 업로드할 수 있다.

## 가정 (Assumptions)

- 사용자는 이미 PDF 형태의 포트폴리오를 보유하고 있다.
- AI 분석 품질은 CLOVA Studio HCX-005 성능에 의존하며, 별도 파인튜닝은 MVP 범위 밖이다.
- 이미지 스캔 PDF는 OCR 없이 처리하지 않는다 (텍스트 추출 불가 케이스로 안내).
- MVP는 팀원 시연 목적이므로 대규모 트래픽 대응은 고려하지 않는다.
- 신고 처리는 운영자(1인)가 직접 DB 조작으로 수행한다.
