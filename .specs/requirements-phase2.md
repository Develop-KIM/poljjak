# 요구사항: 폴짝 2차 개발

## 개요

실 서비스 출시(1~2달 내)를 앞두고 커뮤니티 리텐션과 운영 지속성을 위한 2개 기능을 추가한다.
사용자가 좋은 게시글을 저장하고 재방문할 수 있는 북마크와,
신고·콘텐츠·사용자·AI를 통합 관리할 수 있는 관리자 도구를 갖춘다.

---

## Feature 1: 북마크 (게시글 저장)

### 사용자 스토리

- As a 커뮤니티 사용자, I want to 나중에 다시 보고 싶은 게시글을 저장해두고 싶다,
  So that 스크롤을 다시 올리거나 검색하지 않아도 바로 찾을 수 있다.

### 기능 요구사항

- FR-1: 게시글 상세·목록 카드에 북마크 아이콘(토글). 로그인 필수, 비로그인 클릭 시 로그인 모달
- FR-2: 마이페이지에 "저장한 글" 탭 추가 (기존 분석/내 글/댓글/좋아요 탭 옆)
- FR-3: 저장한 글 목록은 저장 최신순 정렬. 게시글이 삭제됐으면 "삭제된 게시글" 처리 (북마크 row 유지)
- FR-4: 동일 게시글 재클릭 시 북마크 해제

### 비기능 요구사항

- NFR-1 (성능): Optimistic UI — 서버 응답 전에 아이콘 즉시 토글
- NFR-2 (DB): `bookmarks` 테이블 신규 생성 (postId, userId, createdAt). UNIQUE(postId, userId)

### 비목표

- 분석 결과 북마크
- 폴더·태그로 북마크 분류
- 북마크 공개 (본인만 열람)

---

## Feature 2: 관리자 페이지

### 사용자 스토리

- As a 운영자, I want to 서비스 현황과 AI 상태를 한 화면에서 파악하고 싶다,
  So that 이상 징후를 빠르게 감지할 수 있다.
- As a 운영자, I want to 신고된 콘텐츠를 검토하고 처리하고 싶다,
  So that 위반 콘텐츠를 빠르게 제거할 수 있다.
- As a 운영자, I want to 문제 사용자를 정지시키고 필요 시 해제할 수 있다,
  So that 커뮤니티 안전을 유지할 수 있다.
- As a 운영자, I want to AI 프롬프트를 직접 수정하고 이전 버전으로 복원할 수 있다,
  So that 분석 품질을 코드 배포 없이 즉시 개선할 수 있다.

### 탭 구조

| 탭 | 주요 내용 |
|---|---|
| 대시보드 | 서비스 현황 + AI 현황 + AI 모니터링 차트 (1화면) |
| 프롬프트 관리 | 3종 프롬프트 편집·저장·버전 이력·복원·테스트 |
| 신고 관리 | 신고 목록·처리 + 콘텐츠 직접 삭제 |
| 사용자 관리 | 사용자 검색 + role 변경 + 정지·해제 |

### 기능 요구사항

#### 공통
- FR-1: `/admin` 경로. `users.role = 'admin'`인 계정만 접근. 그 외 403 리다이렉트
- FR-2: 헤더에서 admin role 사용자에게만 "관리자" 링크 노출

#### 대시보드 탭
- FR-3: **서비스 현황** 섹션 — 숫자 카드 5개
  - 오늘 신규 가입자 / 오늘 게시글 / 오늘 댓글 / 누적 사용자 / 누적 게시글
- FR-4: **AI 현황** 섹션 — 숫자 카드 6개
  - 오늘 분석 요청 / 성공 건수 / 실패 건수 / 성공률(%) / 평균 응답 시간(초) / 오늘 토큰 사용량 / 누적 토큰 사용량
- FR-5: **AI 모니터링** 섹션 — 차트·목록 3가지
  - 시간대별 분석 요청 분포 막대 차트 (0~23시)
  - 실패한 분석 목록 (사용자 닉네임 / 요청 시각 / pdfUrl) + **재시도** 버튼
  - 최근 7일 일별 평균 응답 시간 꺾은선 차트

#### 프롬프트 관리 탭
- FR-6: developer / designer / general 3종 탭 전환. 현재 활성 프롬프트를 텍스트 에디터로 편집
- FR-7: "저장" 시 신규 버전 row 삽입 + 기존 active → false 전환. 버전 이력 목록에 저장 일시 표시
- FR-8: 이전 버전 선택 후 "이 버전으로 복원" → 해당 버전이 active로 전환
- FR-9: "테스트 실행" — 샘플 텍스트(직접 입력)로 현재 에디터 내용 기준 CLOVA 분석 실행. 결과를 팝업으로 미리보기 (저장 없이)
- FR-10: 저장된 활성 프롬프트를 CLOVA 분석 시 즉시 사용 (파일 대신 DB 조회)

#### 신고 관리 탭
- FR-11: 신고 목록 — 미처리/처리됨 필터, 신고 사유·대상 콘텐츠 미리보기
- FR-12: "콘텐츠 삭제" → 게시글·댓글 소프트 삭제 + 신고 처리됨 표시
- FR-13: 신고와 무관한 직접 삭제 — 게시글·댓글 ID 또는 키워드로 검색 후 삭제

#### 사용자 관리 탭
- FR-14: 닉네임·이메일 검색. 목록에 **가입일 + 마지막 로그인** 표시
- FR-15: role 변경 (user ↔ admin) 토글
- FR-16: 정지/정지 해제 토글 (`suspended_at` 기록)
- FR-17: 정지된 사용자는 로그인 시 "정지된 계정입니다" 안내 후 서비스 이용 불가

### 비기능 요구사항

- NFR-1 (보안): 모든 `/api/admin/*` 라우트에 `requireAdmin(event)` 헬퍼 적용
- NFR-2 (UI): 별도 디자인 시스템 불필요. 기존 컴포넌트 재사용. 별도 admin 레이아웃 사용
- NFR-3 (DB 신규·변경):
  - `users`: `role enum('user','admin') default 'user'` + `suspended_at timestamp` 추가
  - `reports`: `status enum('pending','resolved') default 'pending'` + `resolved_at timestamp` 추가
  - `analyses`: `token_usage integer` 추가 (CLOVA 응답 토큰 수 저장)
  - `bookmarks`: 신규 (`id, post_id, user_id, created_at`, UNIQUE(post_id, user_id))
  - `prompts`: 신규 (`id, job_type enum('developer','designer','general'), content text, is_active boolean, created_by uuid, created_at`)
- NFR-4 (성능): 대시보드 지표는 단순 COUNT 쿼리, 캐싱 불필요 (초기 규모)
- NFR-5: 프롬프트 초기 데이터 — 기존 `server/prompts/*.txt` 파일 내용을 DB 시드로 1회 삽입

### 비목표

- 권한 등급 세분화 (슈퍼어드민 등)
- 자동 제재 (AI 필터링)
- 이메일·슬랙 알림 발송
- 광고·수익 지표
- 정지 계정 게시글 자동 숨김 (수동 삭제만)

---

## 가정 (Assumptions)

- 관리자 계정은 DB에서 직접 `role = 'admin'`으로 설정
- 정지는 기간 구분 없이 운영자가 수동 해제할 때까지 유지
- 프롬프트 테스트 실행 결과는 DB에 저장하지 않음 (미리보기 전용)
- 북마크는 게시글 전용

## 미해결 질문 (Open Questions)

- 실패한 분석 재시도 시 기존 analyses row를 업데이트할지, 신규 row를 생성할지
- 차트 라이브러리 선택 (Chart.js vs 직접 SVG)
