# 스펙: 온보딩

> 기반 스펙: `001-pojjak-mvp.md`, `003-database-schema.md`

## 목표

카카오 OAuth 로그인 직후 신규 사용자 또는 재가입 사용자에게 온보딩 화면을 표시한다.
닉네임·직종·프로필 이미지를 설정하고 나면 서비스 본 화면으로 진입한다.

---

## 온보딩 진입 조건

| 조건                                        | 처리                                                            |
| ------------------------------------------- | --------------------------------------------------------------- |
| `users.onboarding_completed_at IS NULL`     | `/onboarding` 으로 리다이렉트                                   |
| `users.onboarding_completed_at IS NOT NULL` | 이전 방문 경로 또는 홈으로 이동                                 |
| 탈퇴 신청 후 30일 미경과 재로그인           | 계정 복원 + `onboarding_completed_at = null` 리셋 → 온보딩 진행 |
| 탈퇴 30일 경과 후 재가입                    | 신규 계정으로 온보딩 진행                                       |

> **탈퇴 후 재가입은 항상 온보딩을 거친다.**
> 계정 복원 시 `onboarding_completed_at` 을 null로 리셋해 온보딩 흐름을 다시 실행한다.
> 기존 분석 기록·게시글 등 데이터는 복원된다.

**구현 방식**: `app/middleware/onboarding.ts` 에서
`onboarding_completed_at` 확인 후 미완료 시 `/onboarding` 으로 redirect.

---

## 온보딩 흐름 (2단계)

```
[카카오 로그인 완료]
        ↓
[Step 1] 프로필 설정
  - 프로필 이미지 (기본 이미지 제공, 변경 가능)
  - 닉네임 입력 + 중복 체크
        ↓
[Step 2] 직종 선택
  - 개발자 / 디자이너 중 1개 선택
  - "나중에 설정" 건너뛰기 가능
        ↓
[완료] onboarding_completed_at 기록 → 홈 이동
```

## 직종 목록

| value       | label    | 설명                                        |
| ----------- | -------- | ------------------------------------------- |
| `developer` | 개발자   | 프론트엔드·백엔드·풀스택·모바일·데브옵스 등 |
| `designer`  | 디자이너 | UI/UX, 그래픽, 제품 디자이너 등             |

> 마이페이지에서 언제든지 변경 가능.
> 직종 미선택("나중에 설정")으로 완료해도 됨.

## DB 변경 사항

`users` 테이블 컬럼 (`003-database-schema.md` 반영).

| 컬럼                      | 타입                                  | 설명                   |
| ------------------------- | ------------------------------------- | ---------------------- |
| `job_type`                | enum('developer','designer') nullable | 온보딩 Step 2에서 설정 |
| `onboarding_completed_at` | timestamp nullable                    | null이면 온보딩 미완료 |

## 닉네임 규칙

| 항목        | 규칙                                   |
| ----------- | -------------------------------------- |
| 길이        | 2자 이상, 15자 이하                    |
| 허용 문자   | 한글(가-힣), 영문(a-z, A-Z), 숫자(0-9) |
| 금지 문자   | 특수문자, 공백 전부 금지               |
| 중복        | 다른 사용자가 사용 중이면 사용 불가    |
| 검증 정규식 | `^[가-힣a-zA-Z0-9]{2,15}$`             |

### 닉네임 중복 체크 API

```
GET /api/users/check-nickname?nickname={nickname}

응답 200: { data: { available: true } }
응답 200: { data: { available: false } }
```

- 입력 후 500ms 디바운스로 자동 체크
- 체크 중: 로딩 스피너 표시
- 사용 가능: 초록색 체크 + "사용할 수 있는 닉네임이에요"
- 중복: 빨간색 X + "이미 사용 중인 닉네임이에요"
- 형식 오류: "한글·영문·숫자만 2~15자로 입력해주세요"

## 화면 구성

### Step 1 — 프로필 설정

- 상단 진행 표시: `1 / 2`
- 프로필 이미지 원형 영역 (클릭 → 파일 선택)
  - 기본값: `/images/default-avatar.svg`
  - 선택 시 미리보기
- 닉네임 입력 필드
  - 카카오 닉네임 pre-fill
  - 500ms 디바운스 자동 중복 체크
  - 체크 결과 인라인 표시 (로딩 / 사용 가능 / 중복 / 형식 오류)
  - 글자 수 표시
- 다음 버튼: 닉네임 유효 + 중복 아닐 때만 활성

### Step 2 — 직종 선택

- 상단 진행 표시: `2 / 2`
- 제목: "어떤 분야에서 일하고 계신가요?"
- 부제: "포트폴리오 분석에 더 맞는 피드백을 드릴게요."
- 카드 2개 (개발자 / 디자이너) 가로 배치
- 선택 시 카드 강조
- "시작하기" 버튼 (직종 미선택 시에도 활성)
- "나중에 설정할게요" 텍스트 링크

---

## 인수 기준

- AC-1: 신규 로그인 사용자는 `/onboarding` 으로 자동 리다이렉트된다.
- AC-2: `onboarding_completed_at` 이 설정된 사용자는 온보딩을 건너뛴다.
- AC-3: 닉네임이 `^[가-힣a-zA-Z0-9]{2,15}$` 패턴을 통과하지 않으면 다음 버튼이 비활성화된다.
- AC-4: 닉네임 중복 시 인라인 에러가 표시되고 다음 버튼이 비활성화된다.
- AC-5: Step 2에서 직종 미선택 후 "나중에 설정"으로도 완료 처리된다.
- AC-6: 완료 시 `onboarding_completed_at` 이 기록되고 홈으로 이동한다.
- AC-7: 마이페이지에서 닉네임·직종·프로필 이미지를 다시 변경할 수 있다.
- AC-8: 프로필 이미지 10MB 초과 시 인라인 에러가 표시된다.
