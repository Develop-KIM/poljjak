# 요구사항: 공개 갤러리

## 개요

폴짝 공개 갤러리는 유저가 공개로 전환한 포트폴리오 분석 결과를 카드 형태로 모아 보여주는 페이지다. 취업 준비 중인 개발자가 타인의 분석 피드백을 참고하거나, 서비스 첫 방문자가 분석 결과물을 미리 체험하는 두 가지 목적을 동시에 충족한다. 기존 `isPublic` 토글을 갤러리 노출 조건으로 재활용해 별도 설정 없이 자연스럽게 기여된다.

## 사용자 스토리

- As a 취업 준비 중인 프론트엔드 개발자, I want to 갤러리에서 다른 사람의 포트폴리오 분석 결과를 직군별로 필터링해서 보고 싶다, So that 내 포트폴리오에서 개선할 인사이트를 얻을 수 있다.
- As a 서비스 첫 방문자, I want to 로그인 없이 갤러리를 둘러보며 실제 분석 결과물을 확인하고 싶다, So that 서비스 품질을 미리 파악하고 가입 여부를 결정할 수 있다.
- As a 분석을 완료한 유저, I want to 내 분석 결과를 공개로 전환해 갤러리에 자동 노출시키고 싶다, So that 커뮤니티에 기여하면서 내 포트폴리오를 간접 홍보할 수 있다.

## 기능 요구사항

### 갤러리 페이지 (`/gallery`)

- FR-1: 로그인 없이 접근 가능하다. (비회원 열람 허용)
- FR-2: `isPublic = true`인 분석 결과를 최신순으로 무한 스크롤로 표시한다.
- FR-3: 직군(프론트엔드·백엔드·풀스택·DevOps·ML/AI·미지정) 필터를 제공한다.
- FR-4: 연차(신입·주니어·시니어·미지정) 필터를 제공한다.
- FR-5: 최신순 / 인기순(공개 전환 후 조회수 기준, 없으면 생성일 기준) 정렬을 제공한다.
- FR-6: 카드를 클릭하면 기존 공유 페이지(`/analysis/share/:token`)로 이동한다.
- FR-7: 무한 스크롤 — 페이지 하단 도달 시 다음 20개를 추가 로드한다.

### 갤러리 카드

- FR-8: PDF 첫 페이지 썸네일을 카드 상단에 표시한다. (썸네일 생성 불가 시 placeholder)
- FR-9: 직군·연차 뱃지를 표시한다. (없으면 미노출)
- FR-10: AI 총평(`result.summary`) 첫 2줄을 말줄임 처리해 미리보기로 표시한다.
- FR-11: 이슈 개수(high/medium/low 합산)를 표시한다.

### 진입점

- FR-12: 상단 네비게이션에 "갤러리" 메뉴를 추가한다.
- FR-13: 랜딩 페이지(`/`)에 갤러리 프리뷰 섹션(최신 6개 카드)을 추가한다.

### 갤러리 노출 조건

- FR-14: 분석 결과 페이지에서 `isPublic = true`로 전환하면 자동으로 갤러리에 노출된다.
- FR-15: 공개 전환 시 기존 PII 마스킹 모달 흐름을 그대로 거친다. (변경 없음)
- FR-16: `isPublic = false`로 되돌리면 갤러리에서 즉시 사라진다.

## 비기능 요구사항

- NFR-1 (성능): 갤러리 첫 로드는 2초 이내. 카드 20개 기준 API 응답 500ms 이내.
- NFR-2 (보안): 공유 토큰(`shareToken`)이 null인 분석은 갤러리에 노출하지 않는다.
- NFR-3 (접근성): 비회원도 열람 가능 — 서버 라우트에서 인증 불필요.
- NFR-4 (반응형): 데스크탑 3열 · 태블릿 2열 · 모바일 1열 그리드.

## 비목표 (NOT to build)

- 갤러리 전용 "좋아요" 기능 (커뮤니티 좋아요와 별개 구현 안 함)
- 갤러리 내 검색 (키워드 검색)
- 갤러리 노출 여부를 isPublic과 별도로 제어하는 옵션
- 조회수 카운트 기능 (인기순 정렬은 생성일 기준으로 대체)
- PDF 썸네일 서버사이드 생성 (클라이언트에서 vue-pdf-embed로 렌더링)
- 댓글·반응 기능
- 갤러리 카드에서 직접 분석 요청

## 가정 (Assumptions)

- `analyses` 테이블의 `isPublic`, `shareToken`, `result`, `issues`, `pdfUrl`, `jobRole`, `seniority` 컬럼은 이미 존재한다.
- PDF 썸네일은 클라이언트에서 `vue-pdf-embed`로 첫 페이지만 렌더링해 표시한다.
- 인기순 정렬은 별도 조회수 집계 없이 `createdAt` DESC로 대체한다.
- 갤러리 API는 인증 없이 호출 가능하다.

## 미해결 질문 (Open Questions)

- PDF 썸네일을 카드에서 실시간 렌더링하면 성능 이슈가 생길 수 있음 — 카드가 많아질 경우 썸네일 지연 로딩(Intersection Observer) 적용 필요.
- 인기순 정렬을 나중에 실제 조회수 기반으로 바꿀 것인지 결정 필요.
