# 스펙: 디자인 시스템 및 와이어프레임

> 기반 스펙: `001-pojjak-mvp.md`, `requirements-pojjak-mvp.md`

## 목표

폴짝 MVP의 핵심 흐름인 PDF 포트폴리오 분석, 분석 결과 확인, 커뮤니티 피드백, 프로젝트·스터디 모집, 1:1 DM이 하나의 제품처럼 보이도록 디자인 시스템과 주요 화면 와이어프레임을 확정한다.

---

## 디자인 원칙

### 제품 톤

- 토스 스타일의 밝고 미니멀한 인터페이스를 따른다.
- 포트폴리오 분석 서비스이므로 가볍기만 한 톤보다 신뢰감과 실행 가능성을 우선한다.
- 첫 화면은 마케팅 랜딩보다 사용자가 바로 행동할 수 있는 제품 화면에 가깝게 구성한다.
- 한국어 UI만 지원하며, 문구는 짧고 직접적으로 작성한다.

### 핵심 감정

- 막막한 포트폴리오를 구조화해서 다음 행동으로 이어지게 한다.
- AI 피드백과 사람 피드백이 자연스럽게 연결된다는 인상을 준다.
- 초보 사용자도 업로드 제한, 분석 진행, 공유 상태를 쉽게 이해해야 한다.

### 피해야 할 것

- 과한 그라데이션 배경
- 장식용 카드 남발
- 랜딩페이지식 긴 히어로만 있는 첫 화면
- 리치 에디터처럼 보이는 복잡한 작성 UI
- MVP 비목표인 결제, 단체 채팅, 다국어, 관리자 화면 암시

## 비로그인 무료 체험 정책

### 정책 요약

- 비로그인 사용자는 브라우저당 1회 무료 AI 분석을 체험할 수 있다.
- 1회 사용 여부는 `localStorage` 키 `poljjak_guest_used`로 추적한다.
- 1회 소진 후 분석 시도 시 로그인 모달을 표시한다.
- 로그인 유저에게는 이 제한이 적용되지 않는다.

### UI 노출 원칙

- 홈 화면 CTA 아래에 `로그인 없이 1회 무료로 체험할 수 있어요` 문구를 표시한다.
- 분석 시작 버튼 클릭 시 제한 여부를 확인한다. 제한이 걸리면 분석 폼 제출을 막고 LoginModal을 띄운다.
- LoginModal 내 문구: `로그인하고 계속 분석하기` / `비로그인 체험은 1회만 가능해요. 로그인하면 무제한으로 분석할 수 있어요.`

## 디자인 토큰

### 색상

`app/assets/css/main.css`의 CSS 변수 구조를 기준으로 한다. 현재 다음 값이 구현되어 있다.

| 토큰                 | 용도                    | 현재값 (HSL) |
| -------------------- | ----------------------- | ------------ |
| `background`         | 전체 배경               | 210 40% 99%  |
| `foreground`         | 기본 텍스트             | 222 47% 11%  |
| `primary`            | 주요 CTA, 활성 상태     | 217 91% 55%  |
| `primary-foreground` | primary 위 텍스트       | 0 0% 100%    |
| `secondary`          | 보조 버튼, 연한 영역    | 214 32% 95%  |
| `muted`              | 비활성 배경             | 214 32% 94%  |
| `muted-foreground`   | 보조 텍스트             | 215 16% 47%  |
| `accent`             | AI 분석 강조, 선택 상태 | 213 100% 96% |
| `destructive`        | 삭제, 실패, 신고        | 0 84% 60%    |
| `border`             | 구분선                  | 214 32% 90%  |
| `ring`               | 포커스                  | 217 91% 55%  |

권장 의미 색상:

- 성공: 분석 완료, 저장 완료 → `emerald-500`
- 경고: 용량 제한, 페이지 제한 → `amber-500`
- 오류: 파일 형식 오류, 분석 실패 → `destructive`
- 정보: AI 분석 진행, 공유 링크 안내 → `primary`

### 타이포그래피

기본 폰트는 Pretendard (CDN, `app/assets/css/main.css`에 이미 적용됨).

| 이름    | 용도            | font-size / line-height |
| ------- | --------------- | ----------------------- |
| Display | 홈 주요 문구    | 48px / 58px             |
| H1      | 페이지 제목     | 32px / 40px             |
| H2      | 섹션 제목       | 24px / 32px             |
| H3      | 카드 제목       | 18px / 28px             |
| Body    | 본문            | 16px / 26px             |
| Small   | 보조 정보       | 14px / 22px             |
| Caption | 제한 조건, 메타 | 12px / 18px             |

규칙:

- `font-family`는 `Pretendard`, `system-ui`, `sans-serif` 순서로 지정한다.
- 버튼 안 문구는 1줄을 원칙으로 한다.
- 긴 한국어 문장은 최대 너비를 두어 줄 길이가 과도해지지 않게 한다.

### 간격 / 형태

- 페이지 최대 너비: `1120px`
- 읽기 콘텐츠 최대 너비: `760px`
- 카드 radius: `8px`
- 버튼 radius: `8px`
- 모달 radius: `16px`
- 입력 필드 height: `44px`
- 주요 CTA height: `48px`
- 모바일 좌우 padding: `20px`
- 데스크탑 좌우 padding: `32px`

### 그림자

- 기본 UI는 border 중심으로 구성한다.
- 그림자는 모달, 드롭다운, 토스트처럼 레이어가 떠야 하는 요소에만 사용한다.

- `PdfUploadDropzone`
  - PDF 전용, 10MB 이하, 50페이지 이하

  - 스트리밍 분석 중 빈 화면 방지

- `AnalysisScoreCard`
  - 항목명, 점수, 코멘트 표시
- `BeforeAfterBlock`
  - 개선 전 문장과 개선 후 문장을 나란히 표시
- `ShareLinkDialog`
  - 공유 링크 생성 및 복사 UI
  - 분석 결과 공유, 게시글 공유 겸용
- `PostCard`
  - 카테고리, 제목, 본문 일부, 작성자, 댓글 수, 좋아요 수
- `PostImageGrid`
  - 최대 10장, jpg/png/webp/gif 형식
  - 5MB 초과 시 개별 파일 에러 메시지
- `CommentComposer`
  - 로그인 필요 상태에서 LoginModal 트리거
- `ReportDialog`
  - 신고 사유 선택 + 접수 CTA
  - MVP에서는 접수 내용을 DB에 저장하고, 처리 UI는 관리자 페이지 구현 시 추가
- `NotificationPopover`
  - 댓글, 좋아요, DM 알림
  - 읽지 않은 뱃지
- `ChatMessageBubble`
  - 내 메시지, 상대 메시지, 삭제된 메시지 상태

## 주요 화면 와이어프레임

### 1. 홈 (랜딩)

목표:

- 사용자가 바로 PDF 분석을 시작할 수 있어야 한다.
- 비로그인 1회 무료 체험 정책을 명확하게 안내한다.
- 커뮤니티로 진입할 수 있어야 한다.

구성:

- 헤더: 로고, 커뮤니티, 마이페이지, 로그인/시작하기
- 히어로 (center-aligned):
  - 배지: `AI 포트폴리오 분석 · 무료 체험 가능`
  - H1: 두 줄 핵심 문구
  - 설명 텍스트
  - CTA 버튼 2개: `지금 분석 시작하기`, `커뮤니티 둘러보기`
  - 캡션: `로그인 없이 1회 무료로 체험할 수 있어요`
- 기능 하이라이트 3열: 항목별 점수, Before/After, 커뮤니티 연결
- 최근 커뮤니티 3열 그리드

### 2. 로그인 모달 (LoginModal)

목표:

- 비로그인 사용자의 행동 흐름을 끊지 않고 로그인으로 전환한다.
- 비로그인 1회 한도 소진 시에도 동일 모달 사용.

구성:

- 문맥별 제목 prop
  - 예: `로그인하고 계속 분석하기`
  - 예: `로그인하고 좋아요 누르기`
  - 예: `로그인하고 댓글 달기`
- 부제: 한도 소진 시 `비로그인 체험은 1회만 가능해요. 로그인하면 무제한으로 분석할 수 있어요.`
- 카카오 로그인 버튼
- 닫기 버튼

### 3. PDF 업로드 / 분석 요청

목표:

- 사용자가 파일 제한을 업로드 전에 이해해야 한다.
- 추가 요청사항을 선택적으로 입력할 수 있어야 한다.
- 비로그인 한도 소진 시 분석 시작 버튼 클릭 시 LoginModal을 띄운다.

구성:

- 페이지 제목: `포트폴리오 분석하기`
- 드래그앤드롭 업로드 영역
- 제한 조건: PDF, 10MB 이하, 50페이지 이하, 텍스트 선택 가능
- 인라인 에러 케이스:
  - PDF 아닌 파일: `PDF 파일만 가능합니다`
  - 10MB 초과: `파일 크기가 10MB를 초과해요`
  - 51페이지 이상: `최대 50페이지 PDF만 분석할 수 있어요`
  - 스캔 PDF: `텍스트를 추출할 수 없어요. 텍스트 선택이 가능한 PDF를 올려주세요`
- 추가 요청 textarea
- 분석 시작 CTA
- 비로그인 한도 소진 시 → LoginModal (페이지 이동 없음)

### 4. 분석 진행

목표:

- CLOVA Studio 스트리밍 결과를 기다리는 동안 빈 화면을 만들지 않는다.

구성:

- 진행 상태 타이틀
- 단계 표시: 업로드 확인 → 텍스트 추출 → AI 분석 → 결과 정리
- 스트리밍 텍스트 영역
- 취소 또는 뒤로가기는 MVP에서 제공하지 않는다.

### 5. 분석 결과 상세

목표:

- AI 분석 결과를 이해하고 다음 행동으로 이어지게 한다.

구성:

- 결과 헤더: 제목, 생성일, 공개 상태 배지
  - 기본값: **비공개**. 사용자가 명시적으로 공유 링크를 생성해야 공개 상태로 전환된다.
- 항목별 점수 카드 그리드
- 종합 피드백
- Before/After 제안 리스트
- 액션 바: 공유 링크 생성 (ShareLinkDialog), 비공개 전환, 커뮤니티 공유

### 6. 마이페이지 분석 목록

목표:

- 이전 분석 결과를 다시 열람한다.
- 프로필 사진을 변경하고 탈퇴를 신청할 수 있어야 한다.

구성:

- 프로필 영역:
  - 프로필 사진 (클릭 시 파일 선택 → 업로드)
  - 닉네임
  - 이메일 (카카오 제공)
- 분석 결과 목록 (공개/비공개 배지 포함)
- 빈 상태: `아직 분석한 포트폴리오가 없어요`
- 계정 섹션:
  - 탈퇴: `탈퇴 신청` 버튼 클릭 → 확인 Dialog → `30일 후 영구 삭제` 안내

### 7. 커뮤니티 목록

목표:

- 세 게시판을 하나의 탭 UI로 빠르게 탐색한다.

구성:

- 탭: 피드백, 프로젝트 모집, 스터디 모집
- 글쓰기 버튼 (로그인 필요, 비로그인 클릭 시 LoginModal)
- 게시글 카드 리스트
- 피드백 탭: 목록은 비로그인 열람 가능, 상세 진입은 LoginModal
- 프로젝트·스터디 탭: 비로그인 목록·상세 열람 가능, 쓰기만 로그인 필요
- 빈 상태

### 8. 게시글 작성

목표:

- 평문 텍스트 기반의 단순한 작성 경험을 제공한다.

구성:

- 카테고리 선택
- 제목 입력
- 본문 textarea
- 5000자 카운터
- 이미지 업로드 (jpg/png/webp/gif, 최대 10장, 개별 5MB 제한)
- 피드백 탭인 경우 분석 결과 첨부 선택
- 작성 CTA

### 9. 게시글 상세

목표:

- 본문, 이미지, 댓글, 좋아요, DM이 한 화면에서 자연스럽게 이어진다.

구성:

- 제목, 작성자, 생성일, 카테고리
- 본문 (http/https URL은 클릭 가능 링크로 렌더링, XSS 이스케이프 적용)
- 이미지 갤러리
- 좋아요, 댓글 수, 공유 (ShareLinkDialog), 신고 (ReportDialog)
- DM 보내기 (작성자에게 채팅방 생성 또는 기존 방으로 이동)
- 댓글 목록과 댓글 입력 (CommentComposer)

### 10. 채팅

목표:

- 1:1 대화만 단순하게 지원한다.

구성:

- 데스크탑: 좌측 채팅방 목록, 우측 메시지
- 모바일: 목록과 상세를 분리된 화면으로 표시
- 메시지 입력
- 삭제된 메시지 상태

### 11. 알림

목표:

- 댓글, 좋아요, DM 알림을 빠르게 확인한다.

구성:

- 헤더 알림 벨
- 읽지 않은 알림 뱃지
- 드롭다운 목록
- 알림 타입별 아이콘
- 빈 상태

## AI 목업 생성 프롬프트

다음 프롬프트를 v0, Galileo, Uizard 등에 입력해 1차 목업을 만든다.

```text
한국어 웹 SaaS 서비스 "폴짝"의 MVP UI 목업을 만들어줘.

서비스 설명:
개발자와 디자이너가 PDF 포트폴리오를 업로드하면 AI가 분석하고, 분석 결과를 저장·공유하며, 커뮤니티에서 사람 피드백·프로젝트 모집·스터디 모집까지 이어지는 서비스야.
비로그인 사용자는 브라우저당 1회 무료 체험이 가능하고, 이후에는 로그인 유도 모달이 표시돼.

디자인 방향:
- 토스처럼 밝고 미니멀한 UI
- 폰트는 Pretendard 사용
- 신뢰감 있는 블루를 primary color로 사용
- 배경은 거의 흰색에 가까운 neutral tone
- 카드 radius는 8px, 모달 radius는 16px
- 과한 그라데이션, 장식용 배경, 마케팅 랜딩페이지 느낌은 피하기
- 실제 제품 화면처럼 바로 사용할 수 있는 구조
- 한국어 UI
- 데스크탑과 모바일 반응형 고려

필요 화면:
1. 홈: 중앙 정렬 히어로(1회 무료 체험 문구), 기능 3열, 최근 커뮤니티 3열
2. 로그인 모달: 카카오 로그인, 문맥별 CTA (일반 로그인 유도 / 한도 소진 안내)
3. PDF 업로드: 드래그앤드롭, 제한 조건, 스캔 PDF 경고, 추가 요청사항
4. 분석 진행: 단계 표시와 스트리밍 텍스트 영역
5. 분석 결과: 점수 카드, 종합 피드백, Before/After, 공유/비공개 액션
6. 마이페이지: 프로필 사진 수정, 분석 목록, 탈퇴 섹션
7. 커뮤니티 목록: 피드백/프로젝트/스터디 탭, 게시글 카드 3열
8. 게시글 작성: 카테고리, 제목, 본문 textarea, 글자 수 카운터, 이미지 첨부
9. 게시글 상세: 본문, 이미지 갤러리, 좋아요, 댓글, 신고, DM 보내기
10. 채팅: 1:1 채팅방 목록과 메시지 화면
11. 알림: 알림 벨, 읽지 않은 뱃지, 알림 목록
```

## 인수 기준

- AC-1: 디자인 토큰이 `app/assets/css/main.css`와 Tailwind 설정으로 일관되게 사용된다.
- AC-2: 홈, PDF 업로드, 분석 결과, 커뮤니티 목록, 게시글 작성, 게시글 상세의 정적 와이어프레임이 구현 가능할 정도로 명확하다.
- AC-3: 컴포넌트 이름과 책임이 정의되어 중복 구현 없이 확장 가능하다.
- AC-4: 로그인 필요 액션은 페이지 이동이 아니라 LoginModal 패턴으로 정의된다.
- AC-5: PDF 제한 조건(형식, 용량, 페이지, 스캔 여부), 본문 5000자 제한, 이미지 형식·용량·수량 제한이 화면 설계에 포함된다.
- AC-6: MVP 비목표인 결제, 단체 채팅, 리치 에디터, 관리자 화면은 UI에 포함하지 않는다.
- AC-7: 비로그인 사용자의 1회 무료 체험 정책이 홈 문구, 분석 시작 차단, LoginModal 세 곳에서 일관되게 표현된다.
